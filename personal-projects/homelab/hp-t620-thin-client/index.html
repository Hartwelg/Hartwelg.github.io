Thin Clients

I found a couple of thin clients on Ebay to experiment with. One was an HP T620 with an AMD GX-217 Dual-core CPU and 4gb of DDR3 RAM, and the other was a Dell Wyse 5060 with an AMD GX-424CC quad-core CPU and 4gb of DDR3 RAM.

I tried installing Ubuntu 20.04.3 onto the HP T620, but the CPU was not powerful enough to meet the minimum specs required for Ubuntu. Ubuntu requires at least 2.00 GHz per core, and the thin client has 1.67 GHz per core. My next step was to switch to Lubuntu and see if that was usable due to the lower system requirements. This turned up the same result as trying to use Ubuntu, so I changed course to trying to learn about RDP and thin client administration (i.e. deployment of a thin client).

This led me to Open Thinclient, a FOSS operating system for thin clients. There were two goals here:

  • Learn how thin clients work overall. This includes client and server operation and general usage.
  • Use the system as a lite desktop to drive some simple apps that I can use while working, like Spotify and Notepad++.

My first step in learning a bit about how thin clients work was to create a Windows 10 VM on my Optiplex machine, then install Windows 10 IoT Enterprise onto the thin client. I was successful in getting the operating system installed onto the thin client’s 16gb ssd and using remote desktop to connect to the Windows 10 VM installed on the Optiplex. From there, I installed Spotify and Notepad++ and was able to use them both without any problems.

After this little experiment was when I decided to buy the Dell Wyse 5060, with plans to use it as a streaming client in my living room. The plan was to create a rack-mounted gaming pc, and then use the thin client to stream games from the gaming pc to my living room. This plan has not yet come to fruition, but is currently in the works (still gathering parts).
